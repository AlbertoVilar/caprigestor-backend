## 1. Purpose of this guide

This guide is designed to speed up user adoption without requiring the full functional documentation first.

Use it when the goal is to:

- train the farm operator or owner;
- support the first week of usage;
- run a controlled pilot;
- align the minimum daily operational routine.

## 2. What a user must learn first

Before expanding usage, the user must learn four essential blocks:

1. access the correct farm;
2. navigate and maintain the herd;
3. record daily events when they actually happen;
4. use the dashboard and alerts as the daily reference point.

## 3. Recommended onboarding flow

### Day 1 - access and navigation

The user should learn how to:

- sign in with email and password;
- access the correct farm;
- understand the areas `Home`, `Goats`, `Health`, `Lactation`, `Inventory`, and `Commercial`;
- return to the farm, herd, and animal detail screens.

### Day 2 - herd and animal detail

The user should learn how to:

- find an animal by registration, name, or filters;
- open the animal detail page;
- read status, history, and available actions;
- verify that the basic registration is correct.

### Day 3 - reproduction and lactation

The user should learn how to:

- register mating events;
- follow pregnancy and related alerts;
- understand dry-off, resume, and lactation opening;
- record milk production without breaking the current flow rules.

### Day 4 - health, inventory, and commercial

The user should learn how to:

- register a health event;
- identify an active withdrawal period;
- register inventory movements;
- use commercial and minimum operational finance routines.

## 4. Recommended daily routine

The ideal daily system routine is:

1. open the farm dashboard;
2. review alerts and operational agenda;
3. check animals with pending actions;
4. register reproduction, health, and milk events on the same day;
5. update inventory and expenses when they happen.

## 5. Recommended weekly routine

Once a week, the responsible user should:

- review active pregnancies;
- review dry or resumable lactations;
- review completed health events and withdrawals;
- review inventory with purchase cost;
- review commercial and monthly operational finance.

## 6. Minimum safe rollout sequence

The safest implementation sequence is:

1. register or review the farm;
2. review the main herd;
3. confirm the current state of priority goats;
4. start recording events from the current day onward;
5. avoid rebuilding the entire historical backlog at once.

## 7. Most common early mistakes

Avoid these mistakes:

- recording events with the wrong date;
- trying to open a new lactation when the cycle rules do not allow it;
- postponing milk, health, or reproduction records for later days;
- using inventory without recording the real entry;
- ignoring dashboard alerts.

## 8. Practical rule for training the team

Always explain the difference between:

- master data: who the animal is and what its history is;
- operational event: what happened today;
- alert: what needs attention;
- rule block: what the system does not allow at that moment.

## 9. Pilot checklist

Before considering the pilot healthy, confirm that the user can:

- sign in without help;
- find a farm and an animal;
- register mating, health, and milk events;
- review farm alerts;
- record an inventory movement;
- register an operational expense;
- understand why an action was blocked.

## 10. When to move to the full manual

After the team masters the initial routine, use the full manual to deepen:

- reproduction and lactation rules;
- operational withdrawal;
- commercial flows;
- operational finance;
- conscious scope limits.

## 11. Final rollout message

CapriGestor / GoatFarm delivers more value when the team uses it as the farm's daily operating system, not as a delayed record repository.

Register events when they happen.  
Use alerts as a reference.  
Keep operational discipline simple and consistent.
